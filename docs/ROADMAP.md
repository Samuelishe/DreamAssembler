# Roadmap

## Первый этап

1. Подготовить структуру решения из `DreamAssembler.Core` и `DreamAssembler.App`.
2. Описать базовую архитектуру и форматы данных в `docs`.
3. Реализовать MVP-генератор на шаблонах и JSON-словарях.
4. Собрать минимальный WPF-интерфейс для запуска генерации.
5. Добавить тестовые данные и fallback-поведение.

## MVP

В MVP входят:

- главное окно WPF;
- выбор режима генерации;
- выбор уровня абсурдности;
- выбор количества результатов;
- генерация списка результатов;
- отдельный режим ассоциативных 2-4-словных фраз;
- копирование выбранного результата;
- очистка истории текущего запуска;
- загрузка JSON-данных;
- fallback-данные и понятное сообщение при проблеме с файлами.

## После MVP

- unit-тесты для `DreamAssembler.Core`;
- manifest набора данных и древовидная структура словарей;
- редактор словарей и шаблонов;
- сохранение пользовательских настроек;
- более богатые правила тегов;
- улучшение связности коротких текстов;
- чистка и тематическая сегментация большого словаря словесных режимов;
- экспорт результатов;
- пользовательские наборы данных;
- расширение категорий и режимов генерации.

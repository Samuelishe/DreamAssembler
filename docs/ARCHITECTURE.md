# Архитектура DreamAssembler

## Слои приложения

### DreamAssembler.Core

Слой бизнес-логики. Не зависит от WPF.

Основные задачи:

- описание моделей генерации;
- загрузка словарей и шаблонов из JSON;
- хранение fallback-данных;
- выбор шаблонов и элементов словаря с учетом весов и уровня абсурдности;
- сборка итогового текста.

### DreamAssembler.App

Слой интерфейса на WPF.

Основные задачи:

- отображение настроек генерации;
- выполнение команд пользователя;
- показ результатов и истории;
- показ понятного статуса, если данные загружены из fallback.

## Основные классы

### Core

- `GenerationMode` - режим генерации.
- `AbsurdityLevel` - уровень странности результата.
- `TextGenerationOptions` - настройки генерации.
- `TextGenerationResult` - результат одной генерации.
- `DictionaryEntry` - запись словаря.
- `TemplateDefinition` - шаблон предложения, короткого текста или идеи.
- `WeightedRandomSelector` - выбор элементов по весу.
- `DictionaryRepository` - загрузка словарей из `Data/Dictionaries`.
- `TemplateRepository` - загрузка шаблонов из `Data/Templates/templates.json`.
- `DataSetManifestRepository` - загрузка manifest набора данных из `Data/data-manifest.json`.
- `AssociationFragmentRepository` - загрузка и фильтрация больших CSV-словарей слов для ассоциативного режима.
- `TemplateEngine` - подстановка значений в шаблон.
- `TextGeneratorService` - главная точка генерации текста, учитывающая slot-требования, теги, compat-теги, анти-повторы, композиционные роли, базовый порядок narrative-фраз внутри `ShortText` и отдельную ассоциативную генерацию 2-4 слов.

### App

- `MainWindow` - главное окно.
- `MainViewModel` - состояние экрана и команды.
- `ResultItemViewModel` - элемент результата в списке.
- `ClipboardService` - копирование текста в буфер обмена.
- `UserSettingsService` - загрузка и сохранение пользовательских настроек.

### DataTools

- `DreamAssembler.DataTools` - консольный инструмент для валидации и статистики наборов данных.

## Поток данных от кнопки "Сгенерировать" до результата

1. Пользователь нажимает кнопку "Сгенерировать".
2. `GenerateCommand` в `MainViewModel` собирает `TextGenerationOptions`.
3. `MainViewModel` вызывает `TextGeneratorService`.
4. `TextGeneratorService` выбирает путь генерации по режиму.
5. Для `Sentence`, `Idea` и `ShortText` сервис выбирает шаблон по режиму и уровню абсурдности.
6. Для `ShortText` сервис дополнительно следит за композиционными ролями шаблонов и их порядком, чтобы не собирать текст из одинаковых meta-ремарок и не путать временную композицию.
7. Для шаблонных режимов для каждой категории из шаблона выбирается запись словаря с учетом веса, тегов, compat-тегов, слота и близости к выбранному уровню.
8. Для режима `До 4 слов пойдет` сервис выбирает существительное и 1-3 прилагательных из отдельного CSV-лексикона, подбирая форму по роду существительного.
9. `TemplateEngine` подставляет найденные значения в шаблон там, где используется шаблонный режим.
10. `MainViewModel` добавляет результат в список выдачи и в историю текущего запуска.
11. Если при генерации или копировании возникает ошибка, `MainViewModel` показывает понятный статус в UI.

## Где лежат словари, шаблоны и настройки

- Словари: `DreamAssembler/Data/Dictionaries/*.json`
- Manifest набора данных: `DreamAssembler/Data/data-manifest.json`
- Шаблоны: `DreamAssembler/Data/Templates/templates.json`
- CSV-словари ассоциативного режима: `DreamAssembler/Data/AssociationWords/Sources/*.csv`
- Пользовательские настройки: `%LocalAppData%/DreamAssembler/settings.json`

## Как расширять генератор

1. Добавить новые JSON-файлы словарей с нужной категорией.
2. Добавить новые шаблоны в `templates.json`.
3. При необходимости добавить новый режим генерации в `GenerationMode`.
4. Для безопасных шаблонных позиций использовать `slot`, а не только общую `category`.
5. Для шаблонов задавать `slotRequirements`, если категория используется в нескольких грамматических ролях.
6. Для шаблонов `ShortText` задавать `compositionRole`, если нужно контролировать структуру текста и частоту meta-фраз.
7. Если одна категория ведет себя по-разному в разных фразах, разводить ее по отдельным slot-подтипам вместо грубой общей формы, как это уже сделано для `emotion`, `twist`, `concept`, `condition` и сложных `place`.
8. Расширить логику весов или тегов в `TextGeneratorService`, не меняя UI-слой.
9. Позже можно добавить редактор словарей как отдельный модуль, не меняя базовую структуру `Core`.
10. Для смысловой совместимости действия и объекта использовать `compat:*`-теги, а не смешивать это с грамматическими slot-ами.
11. Ассоциативный режим можно расширять отдельно от шаблонного генератора: добавлять новые словари слов, фильтры, стоп-листы и тематические поднаборы без вмешательства в `Sentence` / `ShortText` / `Idea`.

# Ideas Backlog

Этот файл хранит идеи, которые еще не оформлены в план работ.

## Идеи по продукту

- режимы генерации можно позже дополнить сценой, описанием мира, диалогом, названием рассказа;
- добавить пользовательские пресеты абсурдности и жанра;
- добавить экспорт результатов в `.txt` и `.md`;
- добавить избранное внутри текущего запуска, а потом и между запусками.
- добавить отдельные тематические подрежимы для `Словосочетание` и `Несколько слов`, например `визуально`, `архитектурно`, `тревожно`, `сказочно`.
- сделать для словесных режимов почти "reader-first" экран, где короткая фраза визуально важнее управляющих панелей и служебных статусов.
- ввести полноценную систему тем оформления: светлые и темные темы, дизайнерские градиенты, палитры и единый набор brush-ресурсов.
- вынести редкие настройки в отдельное вспомогательное меню из кнопки-шестеренки в кастомной шапке окна.
- постепенно собрать curated-набор собственных бесплатных шрифтов для чтения и заголовков, вместо зависимости от системных шрифтов.

## Идеи по качеству генерации

- мягко учитывать совместимость тегов между несколькими категориями, а не только между записью и шаблоном;
- добавить анти-повтор на последние несколько использованных записей;
- ввести дополнительные типы шаблонов для более связных `ShortText`;
- уменьшить частоту повторяющихся служебных фраз и вводных конструкций в `ShortText`;
- ввести более строгие правила совместимости между `action`, `object`, `place`, `condition` и `twist`, чтобы снижать семантически слабые сочетания;
- при росте данных ввести отдельные slot-подтипы или совместимые подгруппы для `action`, чтобы `чинить`, `разыскивать`, `прятать`, `обменивать` не выбирались одинаково для любых объектов;
- отдельно разнести словарные элементы для наречий, оценок состояния и авторских комментариев, чтобы не смешивать их в неестественные сочетания;
- ограничить частоту meta-фраз и авторских комментариев в одном `ShortText`, чтобы текст не звучал как набор одинаковых ремарок;
- позже ввести более безопасные правила для местоименных отсылок и конструкций с "его", "ее", "их";
- позже аккуратно добавить простые русскоязычные морфологические обходы без полноценного движка.
- перейти от слишком общих категорий к безопасным словарным подкатегориям вроде `object_direct`, `emotion_state`, `place_in`, если это даст лучшее качество без сложной морфологии.
- для словесных режимов ввести более умную фильтрацию слов: стоп-листы, тематические поднаборы, blacklist слишком служебных и слишком технических слов.
- для словесных режимов постепенно отделить более образные и визуальные слова от книжных, канцелярских и слишком абстрактных.

## Идеи по данным

- отделить более бытовые наборы данных от сюрреалистичных;
- завести отдельные JSON-наборы под жанры;
- добавить больше категорий `condition`, `emotion`, `concept`, `style`;
- завести отдельный набор "тихого абсурда", где странность не ломает грамматику.
- поверх больших CSV-лексиконов подготовить curated поднаборы для рисования, мира, атмосферы, костюма, архитектуры и предметов.

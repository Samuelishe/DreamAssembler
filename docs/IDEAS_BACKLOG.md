# Ideas Backlog

Этот файл хранит не wishlist "когда-нибудь", а рабочую карту направлений и рисков.

DreamAssembler должен эволюционировать как atmospheric procedural text engine и reader-first instrument, а не как dashboard, KPI-machine или AI-wrapper.

## Уже существующая manifold map

Это важно держать в голове как факт, а не как wishlist:

- archive / transport / bureaucracy;
- airport;
- mall;
- museum;
- hospitality;
- observatory;
- sanatorium;
- hydroelectric;
- weather_systems;
- coastal_fog.

## Core Direction

- развивать атмосферную procedural coherence, а не усложнение ради усложнения;
- усиливать quiet absurdity, symbolic density и manifold identity;
- держать expansion controlled, а не bulk;
- растить manifolds как emotional ecologies;
- удерживать old-core atmosphere как часть identity, но не как скрытый universal baseline;
- использовать diagnostics как observability, а не как scoring engine.

## Short-Term

- audit-assisted generation tuning как нормальное рабочее направление;
- targeted cadence diversification;
- contextual silence autonomy;
- manifold-local pressure differentiation;
- `museum` cadence-autonomy pass;
- controlled `mall` deepening после очередного runtime checkpoint;
- больше silence-heavy и residue-heavy outputs без template explosion;
- stronger phrase gravity для `ShortText`;
- сохранить workflow `samples + report/snapshot/compare + tests/build`.

## Medium-Term

- manifold-local emotional pressure refinement;
- more temporal diversity for `Sentence` and `ShortText`;
- very soft continuity / resurfacing;
- deepening existing manifolds before opening many new ones;
- next-wave expansion only after runtime checkpoint;
- more agent-readable report wording inside existing diagnostics;
- явная строка в report: `This is not a quality score; use with human sample review.`

## Experimental

- soft continuity memory without rigid narrative chain;
- symbolic resurfacing across adjacent outputs;
- phrase-resonance modes for lexical outputs;
- drift-reading behavior in UI;
- contextual silence variants by manifold;
- additional warning phrasing in diagnostics for:
  - universal quiet fallback;
  - static observation fatigue;
  - legacy-heavy drift;
  - weak manifold autonomy;
  - cadence repetition spike;
  - overdominant template.

## Risks

- metrics-driven sterilization;
- endless diagnostics instead of corpus growth;
- hard quotas / artificial equal distribution;
- treating `report` as optimization target;
- over-suppressing old-core atmosphere;
- mistaking short-batch noise for stable runtime truth;
- template explosion instead of real cadence work;
- manifold growth by nouns only, without pressure/pacing identity.

## Dangerous Overengineering Ideas

- dashboard expansion around diagnostics;
- quality scoring engine;
- recommendation-system logic for text outputs;
- enterprise analytics framework;
- hard manifold mode system;
- deterministic routing;
- AI / embeddings / vector DB / cloud features;
- database-first rewrite;
- universal story-engine ambitions.

## Guidance

Хорошая следующая идея обычно выглядит так:

- маленький targeted runtime pass;
- или controlled manifold deepening;
- или небольшой diagnostics wording improvement;
- или curated data growth по одному осмысленному field.

Плохая идея обычно выглядит так:

- большой новый infrastructure layer;
- bulk manifold wave;
- endless balancing by numbers;
- UI polish without reading benefit;
- попытка сделать DreamAssembler "умным" вместо atmospheric.
